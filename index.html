Pipeline

One config. Full execution.

Define your agents, set your pipeline, and Prismatic Engine handles the rest — review chains, stall recovery, and artifact handoff.

1. INPUT (YAML)

# prismatic.yaml
pipeline:
  design: agy
  build: codex
  review: jules
  vision: agy
→

2. ENGINE

Prismatic Engine parses the manifest, splits work across the agent spectrum, and manages all routing and state.

→

3. EXECUTION

AGY designs → Codex builds → Jules reviews → AGY vision-tests. Each agent checks the previous agent's work. Zero single points of failure.

Bolt-On Marketplace

Extend the spectrum.

Community-built modules that plug directly into your engine. One command to install. Zero configuration.

Database Connector

Run SQL agents safely inside sandboxed environments. Postgres, MySQL, SQLite.

$ prismatic install db-postgres

Discord Notify

Route agent completions, alerts, and summaries directly to Discord channels.

$ prismatic install notify-discord

Sandbox Executor

Dockerized code execution. Every agent runs in an isolated container.

$ prismatic install sandbox-docker
