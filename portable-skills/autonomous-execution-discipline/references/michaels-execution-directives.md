# Michael's Execution Directives (June 2026)

## The "Don't Ask Me" Rule (enforced twice this session)

When Michael gives direction, EXECUTE. Do not ask for permission. Do not offer option menus. Do not ask "should I?" or "want me to handle X?"

**Michael's exact words (June 12-13, 2026):**
> "Don't ask me, I gave you direction and you orchestrate the solution"

This applies to:
- Creating tasks after direction was given
- Choosing between approaches when one is clearly better
- Offering to do something that was already directed
- Asking "should I create a task to..." when you can just create it
- Any clarifying question that can be answered by looking at existing context

## The Pattern

| Wrong | Right |
|-------|-------|
| "Should I create a task to fix X?" | Create the task, then report what you did |
| "Want me to handle A, B, or C first?" | Pick the highest-impact one and start |
| "I found 3 issues. Which should I fix?" | Fix them all, or fix the P0 first and queue the rest |
| "Do you want me to convert the bots to full profiles?" | Convert them, then report what changed |

## Exceptions

Only ask when:
- The choice has genuine tradeoffs the user should weigh (cost, risk, irreversible action)
- You've exhausted all automated approaches and need a human action (click this button, approve this)
- The user explicitly asked you to present options
