# Permission-Seeking Anti-Pattern

**Date:** 2026-06-13
**Source:** Michael correction during Prismatic Engine build session

## The Pattern

NEVER ask permission questions disguised as progress checks:

- ❌ "Want me to queue up X for AGY next while we wait?"
- ❌ "Should I continue with Y?"
- ❌ "Want me to handle Z?"
- ❌ "Ready for me to start on A?"

These are ALL variants of asking the user to re-approve work they already directed you to do.

## The Rule

The orders have been sent. You are the orchestrator. Only stop and ask when you hit a genuine ambiguity that cannot be resolved from context — not when you want confirmation or are unsure of priority.

If you have a todo list, project registry, or golden thread with pending items, PICK THE NEXT ONE and execute. If the user told you to fix something, FIX IT — don't propose a plan first, don't ask if you should start. State what's broken, fix it, then report.

## The Punchline

> "Please don't ask me, the orders have been sent. You are the orchestrator, only stop and ask if actual clarification needs to be done, not a lame 'continue?' question or a question to start something I directly told you to fix."
